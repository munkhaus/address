🚀 Getting Started Guide

Tutorial: Set up your development environment and run the OSM Address Finder app

Welcome to OSM Address Finder!

This tutorial will guide you through setting up the development environment and getting the app running on your machine. By the end of this guide, you'll have:

  • ✅ Flutter development environment configured
  • ✅ Project dependencies installed
  • ✅ App running on your device/simulator
  • ✅ Understanding of the project structure
Time to Complete: 15-30 minutes
Skill Level: Beginner
Prerequisites: Basic command line knowledge

Prerequisites

Required Tools

  • Flutter SDK: 3.32.0 or higher
  • Git: For version control
  • IDE: VS Code, Android Studio, or IntelliJ IDEA

Platform-Specific Requirements

  • iOS Development: macOS with Xcode 16.4+
  • Android Development: Android SDK 35.0.1+

Step 1: Install Flutter

If you don't have Flutter installed yet:

Visit flutter.dev/docs/get-started/install and follow the installation guide for your operating system.

Verify your Flutter installation:

flutter --version flutter doctor

The output should show Flutter 3.32.0 or higher. Run flutter doctor to check for any missing dependencies.

Step 2: Clone the Repository

Clone the project repository to your local machine:

# Clone the repository git clone https://github.com/your-username/osm-address-finder.git cd osm-address-finder # Check the current directory structure ls -la

You should see the Flutter project structure with lib/, pubspec.yaml, and other Flutter project files.

Step 3: Install Dependencies

Install all required packages and dependencies:

# Install dependencies flutter pub get # Generate code (Freezed models) dart run build_runner build --delete-conflicting-outputs
What's happening here?
• flutter pub get downloads all packages listed in pubspec.yaml
• build_runner generates immutable data classes using Freezed annotations

Step 4: Set Up Your Device

Choose one option for running the app:

Option A: Android Device/Emulator

# List available devices flutter devices # Start Android emulator (if not running) flutter emulators --launch <emulator_id>

Option B: iOS Simulator (macOS only)

# Open iOS Simulator open -a Simulator # List available simulators xcrun simctl list devices

Step 5: Run the App

Launch the OSM Address Finder app:

# Run in debug mode with hot reload flutter run --hot # Or run in release mode flutter run --release
Expected Result: The app should launch showing an interactive map centered on Denmark with a search bar at the top. You can pan/zoom the map and search for addresses.

Step 6: Test Core Features

Verify the app is working correctly by testing these features:

Map Interaction

  • ✅ Pan the map by dragging
  • ✅ Zoom in/out using pinch gestures or buttons
  • ✅ Double-tap to zoom in

Address Search

  • ✅ Type in the search bar (e.g., "Copenhagen")
  • ✅ Wait for suggestions to appear
  • ✅ Tap a suggestion to navigate to the location
  • ✅ Verify a red marker appears at the selected location

Location Services

  • ✅ Tap the location button (if available)
  • ✅ Grant location permission when prompted
  • ✅ Verify map centers on your current location

Troubleshooting

Having issues? Here are solutions to common problems:

Build Runner Issues

# Clear generated files and rebuild flutter clean flutter pub get dart run build_runner clean dart run build_runner build --delete-conflicting-outputs

iOS Build Issues

# Navigate to iOS directory and install pods cd ios pod install cd .. flutter run

Android Build Issues

  • Verify Android SDK and build tools are updated
  • Check that Android licenses are accepted: flutter doctor --android-licenses
  • Ensure minimum SDK version is 21+

Location Permission Issues

  • iOS: Check Info.plist has location usage descriptions
  • Android: Verify AndroidManifest.xml has location permissions
  • Grant location permissions manually in device settings if needed
